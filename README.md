# parser_boards

Парсер билбордов с сайта [boards.by](https://boards.by).  
Собирает данные об объектах наружной рекламы и экспортирует их в json и xlsx.

---

## Запуск

**1. Установить зависимости**

```bash
pip install -r requirements.txt
```

**2. Запустить парсер**

```bash
python parser.py
```

После выполнения в папке появятся два файла:
- `result.json` — данные в формате JSON
- `result.xlsx` — таблица Excel

---

## Как работает парсер

### 1. Загрузка списка объектов

Парсер обращается к AJAX'у через `boards.by/local/ajax/map.php` и получает полный список билбордов на карте в виде JSON'а.

### 2. Обход карточек

Для каждого объекта из списка парсер открывает (`/banner/<code>/?side=<side>`) и извлекает дополнительные характеристики:
| `size` | Размер `6x3` |
| `lighting` | Наличие подсветки (`true` / `false`) |
| `Material` | Материал плаката |

Чтобы не перегружать сервер(на этом сайте скорее всего нету защиты, но все равно лучше оставить), между запросами выдерживается пауза 0.2 секунды. Уже загруженное кэшируются.

### 3. Нормализация данных

Нормализуем raw дату:

- **`construction_format`** — определяется по `PROPERTY_TYPE_VALUE` из AJAX-ответа: ключевые слова `билборд`, `ситиборд`, `арка`, `мост` и др. Если совпадений нет — `Нетиповые форматы`.
- **`display_type`** — определяется по объединённой строке из `PROPERTY_TYPE_VALUE` и названия объекта (`NAME`). Ищет ключевые слова `призматрон`, `скролл`, `led`, `экран` и др. Если совпадений нет — `Статика`.
- **`size`** — извлекается регулярным выражением, все разделители (`х`, `×`, `*`, `x`) и единицы (`м`) приводятся к формату `ШxВ`, например `6x3`, `1.2x1.8`.

### 4. Экспорт результатов

Итоговый список записывается в:
- `result.json`
- `result.xlsx`

## Настройки

В начале файла `parser.py` можно изменить:

```python
LIMIT = 100  # максимальное количество объектов
REQUEST_DELAY = 0.2  # задержка
```

## Зависимости

- `requests` — HTTP-запросы
- `beautifulsoup4` + `lxml` — парсинг HTML-страниц
- `pandas` + `openpyxl` — экспорт в Excel
